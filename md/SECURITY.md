# Security Policy

## Supported Versions

Security fixes are provided for the latest published release line of `keylab`.

| Version | Supported |
| ------- | --------- |
| latest  | yes       |
| older   | no        |

## Reporting a Vulnerability

If you believe you have found a security vulnerability in `keylab`, please do not open a public issue or pull request.

Report it privately using one of these channels:

- email: `opensource@authdog.com`
- disclosure policy: <https://authdog.com/whitehat>

Please include:

- a clear description of the issue
- affected versions
- reproduction steps or a proof of concept
- impact assessment, if known

## What to Expect

We will review reports as quickly as possible and aim to:

- acknowledge receipt
- validate the issue
- determine severity and affected versions
- prepare and publish a fix when needed

We may contact you for additional details while investigating.

## Disclosure Guidelines

Please give us reasonable time to investigate and remediate the issue before any public disclosure.

Once a fix is available, we may publish:

- a patched release
- release notes describing the fix
- a public advisory when appropriate
